The orac schema is meant for internal orac database connections, by the core Orac Python stack. The idea is that relevant grants are made to the orac schema, such that it can access business logic in the orac_code schema. I'm not sure whether this POLP has been implemented yet. The ora_ha is destined to be implemented via a plugin and should not be referenced, other than to say that it should only be considered, if explicitly requested. The orac_plugin schema, is the go-between schema, for Orac and plugins, where a plugin requires to create it's own schema. The idea is that the plugin will grant access to entrypoints in it's schema, to the plugin_schema, which will allow Orac access to necessary database resident APIs in the plugin schema.
