# Acoustic Echo Cancellation Design

Orac will not implement native acoustic echo cancellation DSP. Orac owns
only the AEC seam: lifecycle, frame routing, backend selection, and the
minimal diagnostics needed to prove that audio reaches the seam. Real
echo cancellation must come from an external audio processing backend.

LiveKit APM is the first real backend. It is optional and disabled by
default; Orac must continue to run without a LiveKit dependency while the
configured backend is `null`.

## Interface

The AEC interface has two audio paths.

- Playback/reference path: far-end audio generated by Orac TTS and sent
  towards the speaker.
- Capture path: near-end microphone audio before VAD and STT processing.

The interface is:

```python
process_reverse_frame(frame: bytes) -> None
process_capture_frame(frame: bytes) -> bytes
reset() -> None
```

`process_reverse_frame()` accepts playback reference frames. These frames
tell a future backend what Orac is sending to the speaker.

`process_capture_frame()` accepts microphone frames and returns the
echo-processed capture frame. The returned bytes must use the same frame
format as the input.

`reset()` clears backend state at turn or session boundaries so stale
reference audio cannot affect later turns.

## Frame Contract

All Orac AEC frames use this exact format:

- 16000 Hz
- 10 ms
- mono
- signed int16
- 160 samples
- 320 bytes

Adapters must reject frames that do not match this contract. The playback
reference path and the capture path both use the same frame size.

## Backend Rules

`NullAcousticEchoCanceller` is pass-through only. It validates frames,
accepts reverse frames, returns capture frames unchanged, and provides no
real echo cancellation. It exists for disabled or no-backend operation.

A real backend must support timing alignment between the playback
reference stream and the capture stream. At minimum, Orac expects a
configurable stream delay in milliseconds. Better backends may later use
measured playback and capture timestamps.

LiveKit APM matches the desired backend shape: reverse stream input,
capture stream processing, fixed 10 ms frames, and configurable stream
delay. The Orac-side implementation adapts LiveKit to the
`AcousticEchoCanceller` interface and does not add Orac-owned echo
suppression logic.

To enable the LiveKit backend, install the optional dependency and set:

```ini
[voice]
playback_backend = native
aec_backend = livekit
aec_stream_delay_ms = 0
```

The native playback backend is required so Orac can feed playback
reference frames into LiveKit's reverse stream.

## Barge-In Implications

Production speaker barge-in requires a real AEC backend or equivalent
operating-system/device echo cancellation. Correct turn lifecycle logic is
necessary, but it is not sufficient: raw VAD or wake-word monitoring can
self-trigger on Orac's own speaker output when echo is not suppressed.

Until a real AEC backend or equivalent audio isolation is active, raw VAD
barge-in remains diagnostic and experimental on ordinary speakers.
